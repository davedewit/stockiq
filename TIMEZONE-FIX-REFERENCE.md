## How It Works

### Data Flow:
1. **RSS Feed** → `pub_date` string (e.g., "Sun, 15 Mar 2026 18:25:43 GMT")
2. **fetch_stock_news()** → Returns `{'date': pub_date, ...}`
3. **Main loop** → Stores as `'timestamp': latest_news.get('date', '')`
4. **Stock page** → Parses timestamp, outputs with `data-timestamp` attribute
5. **news.html** → Parses timestamp, outputs with `data-timestamp` attribute
6. **news.js** → Parses timestamp, outputs with `timestamp` field
7. **JavaScript** → timezone-converter.js converts to viewer's local timezone

### Backend Format (stored in HTML):
```html
<span data-timestamp="2026-03-15T18:25:43+00:00" class="article-date">
  Sun, March 15, 2026 at 6:25 PM UTC 4 min read
</span>
```

### Frontend Display (after JavaScript conversion):
- **Australia (GMT+11):** `Mon, March 16, 2026 at 5:25 AM GMT+11 4 min read`
- **New York (GMT-5):** `Sun, March 15, 2026 at 1:25 PM GMT-5 4 min read`
- **London (GMT+0):** `Sun, March 15, 2026 at 6:25 PM GMT+0 4 min read`

function formatLocalTime(isoString) {
    const date = new Date(isoString);
    // Formats to: "Mon, March 16, 2026 at 5:25 AM GMT+11"
    return `${weekday}, ${month} ${day}, ${year} at ${time} ${offsetStr}`;
}

// Finds all elements with class="article-date" and data-timestamp
document.querySelectorAll('.article-date[data-timestamp]').forEach(el => {
    const timestamp = el.getAttribute('data-timestamp');
    const formattedDate = formatLocalTime(timestamp);
    el.textContent = `${formattedDate} ${readTime}`;
});
